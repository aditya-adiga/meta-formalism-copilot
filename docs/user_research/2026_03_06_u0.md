First user research experiment.

## Setup
I opened the app for a user, and gave them minimal guidance beyond what they've already heard from me about the project in passing.

## Observations

### 1
The user did not go into the process with a document or idea in mind. This made the engagement much more improvisational than most of our expected use cases.

**Recommendation**: make sure users have an idea or document to start from

### 2
the "deductive (lean)" chip being autoselected doesn't make the interface clear

### 3
The input tab is now very busy; it's difficult to read text in the "context" box

### 4
The semiformal and lean code tabs are not hidden by default and should be

### 5
After typing in the context box, refinement chips pop up for the box--this is a weird interaction and takes the input section to being an output step already.

### 6
User tried a "statistical model" for discussion of "do myths make us human" saying "what would that look like?"

### 7
Decompose buttons didn't seem to work

### 8
Formalization has significant wait times; having a time remaining estimate would be nice, should be possible with the analytics. I'd like to capture the analytics data again, maybe it's not a problem to persist that data even (though put it in .gitignore so people aren't submitting their analytic data?)

### 9
different formalisms need an entry point definition of what they are, what contexts they are appropriate for

### 10
User looked up names from the formalisms, inclined to start wikiwalks from it

### 11
User said the results were "excellent backbone for a book"

### 12
New formalism results are not yet exportable

### 13
Each formalism should have a unique interface, needs to be designed

### 14
Would be valuable to be able to 'start a new session' and have dropdowns on every page to return to old sessions, where a new session resets the whole data setup, decomposition graph, which tabs are hidden, etc.
